## 1. 시장성

### 타겟 시장

| 우선순위 | 세그먼트 | 설명 | 규모 |
|----------|----------|------|------|
| 1순위 | 한국 | D-Day 카운트다운 문화가 일상어로 정착. 수능 D-Day, 군 전역, 커플 기념일, 입사일 등 이벤트 카운팅이 생활화. 개발팀 모국어 → 빠른 피드백·ASO 최적화 가능 | 스마트폰 이용자 약 4,500만 |
| 2순위 | 미국 | 영어 UI 이미 지원. 감성적 카운트다운 앱 카테고리 공백 — 기존 앱은 Countdown Widget 등 기능 중심. dayly의 "42번의 아침" 스타일 표현은 미국 시장에서 차별화 포인트로 작동 가능. iOS 비율 57%, App Store 결제력 높음 | 스마트폰 이용자 약 2.8억 |
| 3순위 | 일본 | ja 로컬라이제이션 완료. iOS 점유율 약 70%로 한국보다 높음. LINE(점유율 85%) 공유 최적화 필요. IAP 가격은 Apple 고정 티어(¥480·¥980) 기준 설정 요. 卒業式(3월)·入学式(4월) 등 계절성 이벤트 연계 가능 | 스마트폰 이용자 약 9,000만 |

### 시장 트렌드

- **위젯 문화 확대**: iOS 16+ 잠금화면 위젯, Android 12+ 홈화면 위젯 재활성화로 위젯 앱 전반 수요 급증
- **감성 콘텐츠 소비**: MZ세대 중심으로 날짜·숫자보다 "의미 있는 문구"를 선호하는 트렌드
- **SNS 공유 카드**: 인스타그램·카카오톡 공유용 카드 생성 수요 (생일 카운트, 기념일 등)
- **오프라인 우선**: 서버 없이 동작하는 앱에 대한 개인정보 민감도 우려 감소 효과

### 시장 규모 추정

- Google Play "D-Day" 키워드 앱 다운로드 상위 10개 합산 추정 500만~2,000만 설치
- 한국 앱스토어 라이프스타일 카테고리 상위권 D-Day 앱 평균 MAU 수십만 수준
- 미국 "Countdown" / "Days Until" 키워드: 경쟁 앱 다수지만 감성 카테고리 공백. App Store 미국 결제력은 한국 대비 IAP ARPU 3~5배 수준으로 추정

## 2. 수익 구조

### North Star Metric

**위젯 설치수 (`home_widget_installed`)** — 단일 핵심 지표.

홈화면에 위젯을 박은 유저 = 앱 삭제 마찰 최대 = 리텐션 강점의 직접 측정값. 모든 기능 개선·마케팅 결정은 "이것이 위젯 설치수를 늘리는가?"로 판단한다.

- 이 이벤트는 TODO-BIZ-1(Firebase Analytics)에 이미 리스트업됨. P0 완료 시 즉시 추적 시작.
- 부지표: `share_tapped` (바이럴 입력), `first_widget_created` (획득 품질)

### 현재 구현된 수익원

| 수익원 | 구현 상태 | 예상 단가 |
|--------|-----------|-----------|
| AdMob 배너 광고 | 구현됨 (에뮬레이터에서 비활성화, 실기기 재활성화 필요) | eCPM $0.5~$2 (한국 기준) |
| AdMob App Open 광고 | 구현됨 (쿨다운 로직 적용 후 재활성화 필요) | eCPM $3~$8 |
| 프리미엄 | `isPremium` 필드 존재, **구매 플로우 미구현** | — |

### 프리미엄 가치 제안 — 역바이럴 강화 모델

**워터마크 제거는 프리미엄 혜택이 아니다.** 워터마크는 신규 유저 획득의 핵심 레버이므로 절대 제거하지 않는다.

대신 **역바이럴(Reverse Viral)** 구조를 설계한다: 프리미엄 유저가 공유할 때 더 아름다운 카드가 나가고, 그 카드 자체가 "나는 프리미엄이다"를 보여줌으로써 무료 유저에게도 dayly를 노출시킨다.

| 프리미엄 혜택 | 설명 | 바이럴 효과 |
|---------------|------|-------------|
| 프리미엄 전용 공유 카드 프레임 | 무료보다 더 고급스러운 글래스모피즘/계절 한정 프레임 | 강함 — 카드 수령자가 "이게 뭐야" 반응 유발 |
| 공유 카드 `Premium` 배지 | 카드 하단에 subtle한 "dayly Premium" 표기 | 강함 — 역워터마크처럼 동작 |
| 시즌 한정 테마 팩 | 수능·크리스마스·벚꽃 시즌 전용 테마 | 중간 — 구매 트리거 명확 |
| 위젯 슬롯 무제한 | 무료: 5개, 프리미엄: 무제한 | 없음 |

### 잠재 수익원 (미구현)

| 수익원 | 우선순위 | 예상 단가 |
|--------|----------|-----------|
| 인앱 결제 — 프리미엄 1회성 | 높음 | $2.99 (한국), ¥480 (일본 Apple 티어) |
| 인앱 결제 — 시즌 테마 팩 | 중간 | $0.99~$1.99 per pack |
| 구독 (프리미엄+) | 낮음 | $0.99~$1.99/월 |

### 수익 시뮬레이션 (MAU 10만 기준)

```
광고 수익 (배너 + App Open):
  - DAU 30% = 3만 / 평균 2회 노출
  - App Open eCPM $5 → 월 약 $900
  - 배너 eCPM $1 → 월 약 $600
  합계: 약 $1,500/월 (MAU 10만 기준)

인앱 결제 (프리미엄 구현 시):
  - 전환율 1% × 10만 = 1,000명 × $2.99 = $2,990 (일회성)
  - 이후 신규 유저 전환 지속
```

### 수익 전략 우선순위

**광고 먼저, IAP는 나중에.** 유저 행동 데이터가 없는 상태에서 IAP 가격/전환율을 설정하면 미스매치 리스크가 높다. Analytics로 행동 데이터를 3~4주 확보한 뒤 IAP 가격·트리거 지점을 결정한다.

### 핵심 과제

- **광고 우선 전략**: Firebase Analytics 추가(데이터 수집) → App Open 쿨다운 24시간 구현(광고 수익 즉시 발생) → 배너 광고 실기기 검증 순서로 진행.
- **프리미엄 구매 플로우 구현**: `isPremium` 모델은 준비됨. Analytics 데이터로 전환 지점 파악 후 `in_app_purchase` 연동. **현재 구매 게이팅 없음 — IAP 구현 전까지는 OK, 추후 반드시 추가 필요.**

## 4. 유지율 (Retention)

### 리텐션 강점

| 요소 | 설명 | 영향 |
|------|------|------|
| 홈화면 위젯 | Small/Medium/Large 3가지 크기. Android + iOS 모두 지원. 매일 잠금 해제 시 D-Day 노출 | 매우 강함 — 앱 삭제 마찰 증가 |
| 로컬 알림 | D-7/D-3/D-1/D-Day 알림 자동 예약 | 중간 — 재방문 유도 |
| 마일스톤 기능 | 이벤트에 체크리스트 추가 가능 | 중간 — 콘텐츠 축적 효과 |
| 오프라인 동작 | 서버 없이 완전히 동작 → 로딩 없이 즉시 열림 | 앱 삭제 마찰 감소 |

### 리텐션 약점 / 개선 기회

- **기념일 자동 반복 이벤트 (P1 → 구현 예정)**: D-Day가 지나면 앱 사용 이유가 사라지는 구조적 문제. `DaylyWidgetModel`에 `isRecurring: bool` 필드 추가, WidgetUpdateManager가 매일 자정 체크 시 지난 날짜 + `isRecurring=true`이면 다음 해 같은 월/일로 자동 갱신. 커플 기념일·생일·입사일이 핵심 케이스.
- **소셜 기능 없음**: 친구와 D-Day 공유/함께 보기 등 소셜 루프 없음
- **알림 커스터마이징**: 현재 특정 시점/반복 선택. 더 세밀한 개인화가 가능

## 5. 경쟁 & 차별성

### 주요 경쟁 앱

| 앱 | 플랫폼 | 강점 | 약점 |
|----|--------|------|------|
| D-Day (Chill Chill) | iOS | 한국 시장 1위, 깔끔한 UI | Android 미지원 or 부실 |
| Countdown Widget | iOS/Android | 다양한 위젯 스타일 | 감성 낮음, 광고 많음 |
| Days Until (Widgetsmith 계열) | iOS | 높은 커스터마이징 | 복잡한 설정, 영어 중심 |
| 디데이 (한국 앱 다수) | Android | 스토어 노출 많음 | 구식 UI, 광고 과다 |

### dayly의 포지셔닝

> **"D-Day 앱"이 아니라 "날짜에 이름을 붙이는 앱"이다.**

기존 D-Day 앱은 숫자를 세는 도구다. dayly는 숫자를 감성적 언어로 번역해 날짜에 의미를 부여한다. 이 포지셔닝은 Countdown Widget·Days Until 같은 기능 경쟁 앱과 직접 비교되지 않는 별도 카테고리를 만든다.

- ASO 키워드: "D-Day" (검색 볼륨) + "기념일 위젯" / "anniversary countdown" (전환율) 듀얼 공략
- 스토어 설명 첫 줄: "숫자가 아닌 이야기로 D-Day를 기억하세요"

### dayly의 차별성

1. **감성 우선 UX**: "42번의 아침", "42번의 밤" 등 숫자를 의미 있는 언어로 치환하는 카운트다운 표현 방식 — 경쟁 앱 모방 불가능한 언어 자산
2. **위젯 완성도**: Small/Medium/Large × Android/iOS 총 6종 위젯, 테마 색상 완전 동기화
3. **공유 카드 품질**: 글래스모피즘 UI 기반 고품질 공유 카드 (SNS 공유에 최적화) + 프리미엄 역바이럴 프레임
4. **서버리스 = 개인정보 안심**: Firebase 의존성 제거됨. 모든 데이터 기기 로컬 저장
5. **다국어 완성도**: ko/ja/en 알림 메시지, 위젯 텍스트까지 언어 동기화

### 차별화 유지를 위한 제품 원칙 (CLAUDE.md 기반)

> "위젯을 더 감성적으로, 공유를 더 매력적으로, 정보를 더 명확하게" 만들지 않는 기능은 구현하지 않는다.

## 성장 플라이휠

```
         [앱스토어 검색]
               │
               ▼
        [첫 위젯 생성]
               │
     ┌─────────┴─────────┐
     ▼                   ▼
[홈화면 위젯 설치]   [공유 카드 생성]
     │                   │
     │               [SNS/카톡 공유]
     │                   │
     │            [워터마크 노출]
     │                   │
     │            [신규 유저 유입] ──┐
     │                              │
     ▼                              │
[매일 잠금 해제 시 노출]            │
     │                              │
     ▼                              │
[D-Day 도달]                        │
     │                              │
  isRecurring?                      │
     ├─ Yes → [다음 해 자동 갱신] ──┘
     └─ No  → [새 이벤트 생성 유도]
                   │
            [리텐션 유지]
```

**플라이휠 핵심 레버:**
1. **공유 품질** → 워터마크가 보여야 신규 유입 발생. 프리미엄 유저는 더 아름다운 카드 + Premium 배지로 역바이럴 강화 (워터마크는 절대 제거 불가)
2. **위젯 지속성** → 홈화면에 박힌 위젯이 앱 삭제 마찰 최대화. North Star = `home_widget_installed` 수
3. **기념일 반복** → D-Day 지나도 앱 잔류 → 플라이휠 순환 지속 (`isRecurring` P1 구현 후 활성화)
