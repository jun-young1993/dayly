당신은 10년 차 모바일 플랫폼 아키텍트이자 UX 전략가입니다.
Flutter 기반 D-Day 앱에 iOS 및 Android 홈화면 위젯 기능을 설계하세요.

상황:
- D-Day 아이템이 여러 개 존재
- 사용자가 위젯을 홈화면에 추가할 수 있어야 함
- 로컬 데이터 기반
- 실사용 가능한 설계 필요

다음 질문을 반드시 포함하여 설계하세요:

1️⃣ Step-Back: 위젯의 목적 정의
- 위젯은 빠른 확인용인가?
- 감성적 노출용인가?
- 여러 개를 보여주는 것이 적절한가?

2️⃣ 대표 D-Day 표시 전략 설계
- 자동 선택 (가장 가까운 날짜?)
- 사용자가 대표 선택?
- 여러 위젯 인스턴스 허용?
- 각 전략의 UX 장단점 비교

3️⃣ iOS vs Android 제약 분석
- 위젯에서 좌우 스크롤 가능한가?
- 인터랙션 가능한 범위는 어디까지인가?
- 클릭 시 앱 열기 가능한가?
- 위젯에서 특정 D-Day 상세 화면으로 딥링크 가능한가?
- 플랫폼 정책 위반 요소는 없는가?

4️⃣ UX 설계
- Small / Medium / Large 위젯 구성 제안
- 여러 D-Day를 표현하는 시각적 방법
- 감성형 vs 정보형 레이아웃 비교
- 정보 과부하 방지 전략

5️⃣ 위젯 ↔ 앱 연결 흐름
- 위젯 클릭 시 동작
- 특정 D-Day 클릭 시 딥링크 구조
- 앱 내부에서 “홈화면에 위젯 추가하기” 버튼이 가능한지 여부
- 가능하다면 어떤 방식으로 가이드해야 하는지

6️⃣ Flutter 구현 전략
- 사용 가능한 패키지 예시
- iOS WidgetKit 연동 방식 개요
- Android AppWidget 구조 개요
- 데이터 동기화 방식 (SharedPreferences / Hive / App Group)

7️⃣ 기술적 난이도 및 유지보수 분석
- 구현 난이도 (상/중/하)
- 플랫폼별 개발 비용 차이
- 실무에서 자주 겪는 문제

8️⃣ 최종 추천 전략
- MVP 기준 추천 구조
- 확장 가능한 구조
- 실제 서비스에 가장 적합한 방향

답변은:
- 플랫폼 정책 기반으로 현실적으로 작성
- “가능/불가능”을 명확히 구분
- UX 전략 + 기술 구조를 함께 설명
- 단순 이론이 아닌 실무 기준으로 작성
