v1# Dayly 앱 전체 점검 보고서

> 작성일: 2026-03-05
> 중점: 사업 가치 + 사용자 사용성 + 코드 품질

---

## TL;DR — 한 줄 평가

**아이디어는 좋다 (D-Day + 홈위젯 + 공유). 하지만 핵심 기능이 미완성이고, 데이터 유실 위험이 있으며, 수익화 전략이 없다. 지금 출시하면 1-2달 내 나쁜 리뷰로 손상된다.**

---

## 1. 사업적 관점

### 1.1 핵심 가치 제안 (USP) 검증

| USP | 현재 상태 | 리스크 |
|-----|-----------|--------|
| "홈화면 위젯으로 D-Day 확인" | ⚠️ 미완성 (딥링크 버그, iOS 미검증) | 앱 설명과 실제가 다르면 1별 리뷰 |
| "감성적인 공유 카드" | ✅ 작동함 | 낮음 |
| "알림으로 D-Day 챙기기" | ⚠️ 부분 작동 (64개 제한 미처리) | 알림 안 온다는 리뷰 폭주 |

### 1.2 사용자 유지 (Retention) 위협 요소

#### 🔴 최대 위협: 데이터 유실
- Firebase 인증/DB 비활성화됨 (코드는 있지만 주석 처리)
- 재설치, 기기 변경 시 **모든 D-Day 데이터 소멸**
- 예상 시나리오: 사용자가 소중한 기념일 등록 → 폰 교체 → 데이터 사라짐 → 1별 리뷰
- **사용자 유지율 1개월 내 50% 이탈 예상**

#### 🔴 두 번째 위협: 알림 신뢰성
- `SCHEDULE_EXACT_ALARM` 권한 요청이 첫 위젯 추가 때만 발생
- 거부 후 다시 묻지 않음 → 알림 조용히 작동 안 함
- AlarmManager 64개 제한 처리 코드 없음 (CHANGELOG에만 언급)
- 앱의 핵심 기능인 "챙겨주는 알림"이 무결성 없음

#### 🟡 세 번째 위협: 온보딩 부재
- 앱 첫 실행 시 빈 화면만 표시
- "+" 버튼이 무엇을 하는지 안내 없음
- 신규 사용자 5분 내 이탈율 높을 것으로 예상

### 1.3 수익화 구조 문제

```
현재 수익 구조: 배너 광고 하나
예상 ARPU: $0.001 수준 (배너 CPM $0.5~1 기준)
```

- `isPremium` 플래그가 코드에 있지만 결제 시스템 없음
- 사용자가 `isPremium=true`로 변경 가능 → 워터마크 제거됨 → UI 혼란
- In-app purchase 미구현, 프리미엄 전용 콘텐츠도 없음
- 보상형 광고, 구독형 없음 → 배너만으로는 지속 불가능

### 1.4 시장 포지셔닝 모호성

- CHANGELOG와 주석은 한국어, UI는 영어
- 한국 앱인지 글로벌 앱인지 불명확
- 타겟: 20~30대 감성적 D-Day 관리 → 정의는 됐지만 마케팅 텍스트 없음
- 앱스토어 최적화(ASO) 준비 안 됨

## 2. 사용자 사용성 문제

### 2.1 사용자 여정별 문제점

```
설치 → [문제: 온보딩 없음]
  ↓
D-Day 추가 → [문제: 알림 권한 요청 타이밍 놓침]
  ↓
홈화면 위젯 추가 → [문제: 어떤 D-Day를 표시할지 선택 불가]
  ↓
알림 수신 → [문제: 권한 거부 시 조용히 실패]
  ↓
재설치 → [문제: 모든 데이터 손실]
```

### 2.2 홈화면 위젯 UX 미완성

- Android ConfigActivity는 구현됨
- iOS: AppIntentConfiguration 있지만 Xcode 수동 설정 필요 (문서 없음)
- "어떤 D-Day를 위젯으로?" 선택 UI 불완전
- 위젯 클릭 → 앱 딥링크 → 파싱 로직 버그 가능성 (아래 참고)

### 2.3 마일스톤/메모 기능 반쪽짜리

- 생성 시트에서 추가 불가, 상세 화면에서만 가능
- `DaylyMilestone.dueDate` 필드는 저장되지만 UI에 표시 안 됨
- 메모 텍스트 길이 제한 없음 → UI 레이아웃 깨짐 가능

### 2.4 공유 기능 약점

- 공유 성공 후 피드백 없음 (화면 안 닫힘, 토스트 없음)
- 공유 텍스트가 `'dayly'` 한 단어 → SNS 전파력 없음
- 공유 후 "앱 다운로드 링크" 없음 → 바이럴 마케팅 불가

## 3. 코드 버그 및 기술 부채

### 3.1 버그 목록

| 심각도 | 위치 | 문제 | 해결 방향 |
|--------|------|------|-----------|
| 🔴 | `deep_link_handler.dart` | `uri.host == 'detail'` 검사 오류 가능 | `uri.path.startsWith('/detail/')` 로 변경 |
| 🔴 | `main.dart` | Firebase 초기화 전체 주석 처리 | 재활성화 또는 의도적 비활성화 문서화 |
| 🟡 | `notification_scheduler.dart` | DST 경계에서 타임존 계산 오류 가능 | `tz.TZDateTime.from()` 활용 |
| 🟡 | `notification_scheduler.dart` | AlarmManager 64개 제한 미처리 | 예약 전 pending 수 확인 로직 추가 |
| 🟡 | `share_preview_screen_v2.dart` | 16ms 하드코딩 지연 (프레임 대기 불안정) | `addPostFrameCallback` 으로 교체 |
| 🟡 | `auth_gate.dart` | `print()` 로그가 프로덕션에 노출 | `debugPrint` → Crashlytics로 이전 |

### 3.2 주요 기술 부채

```
Firebase 코드 유산
  ├─ config.dart, auth_gate.dart, firebase_options.dart 존재
  ├─ 하지만 main.dart에서 전부 주석 처리
  └─ 언제 다시 활성화할지 불명확 → 방치될수록 버그 가능성 높아짐

isPremium 결제 스텁
  ├─ DaylyWidgetStyle.isPremium 필드 존재
  ├─ UI에서 토글 가능
  └─ 실제 구매 검증 없음 → 사용자 혼란

enum 직렬화 취약성
  ├─ themePreset.name → 'night' (문자열 저장)
  └─ enum 이름 바꾸면 기존 데이터 전부 깨짐 (마이그레이션 로직 없음)
```

### 3.3 성능 우려

- `SharedPreferences` 호출 시 캐싱 없음 (매번 디스크 읽기)
- 공유 캡처 시 `pixelRatio = 3` 고정 → 저사양 기기 OOM 가능
- 배경 그라데이션 애니메이션이 전체 위젯 트리 rebuild 유발 가능성

## 4. 긍정적 평가 (강점)

- 감성적 디자인 퀄리티 좋음 (글래스모피즘, 5가지 테마)
- 알림 예약 아키텍처는 잘 설계됨 (`NotificationRepository` 패턴)
- LLM 없이 오프라인 문구 생성 (`dayly_sentence_templates.dart`) — 영리함
- 코드 가독성 높음 (풍부한 주석)
- 홈화면 위젯 시도 자체는 차별화 포인트

## 5. 개선 로드맵 (우선순위순)

### Phase 1 — 긴급 수정 (1~2주)
1. **딥링크 파싱 버그 수정** → 홈위젯 클릭 기능 확보
2. **debug print 제거** → 프로덕션 배포 최소 조건
3. **AlarmManager 64개 제한 처리** → CHANGELOG 약속 이행
4. **권한 요청 타이밍 조정** → 앱 첫 실행 시 알림 권한 요청

### Phase 2 — 사용자 유지 (2~4주)
1. **Firebase 재활성화 or 로컬 백업 내보내기/가져오기** → 데이터 유실 방지
2. **온보딩 화면 추가** → 신규 사용자 이탈 방어
3. **홈화면 위젯 iOS 검증** → 핵심 USP 완성
4. **공유 개선** — 문구 자동 생성 + 앱 링크 포함

### Phase 3 — 수익화 (4~6주)
1. **In-app purchase 구현** (`isPremium` 플래그 완성)
2. **프리미엄 가치 정의** (추가 테마, 위젯 무제한, 광고 제거)
3. **보상형 광고 추가** (공유 후 노출)

### Phase 4 — 확장 (6주+)
1. **다국어 지원** (한국어 / 영어 선택)
2. **앱스토어 최적화** (스크린샷, 설명문, 키워드)
3. **카테고리/검색** → 위젯 10개 이상일 때 UX 개선
